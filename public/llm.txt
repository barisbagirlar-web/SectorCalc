# SectorCalc

> Deterministic engineering calculators for shop-floor engineers, estimators, and QA. Client-side only - your data never leaves the browser. Same inputs always produce identical outputs.

This file mirrors `/llms.txt` for discovery paths that expect `llm.txt`. Prefer https://www.sectorcalc.com/llms.txt as the canonical index.

## Architecture (current)

- Theme: `/sc-theme.css` + `/sc-theme.js` (`data-theme`, localStorage `sectorcalc-theme`). `#themeToggle` is click-to-toggle, drag-to-move, double-click to reset (`sectorcalc-theme-pos`).
- Brand wordmarks: `/sectorcalc-logo.png` (light) and `/sectorcalc-logo-dark.png` (dark).
- Report contrast: tokenized report surfaces; SVG/canvas/html2canvas ink from `theme-palette` at render time (no hardcoded light-theme fills). Open reports refresh on theme change.
- Homepage Live Cell hero (`/sc-hero-cell.js`):
  - Self-hosted Three.js at `/vendor/three/` (no unpkg); modulepreload + parallel imports.
  - SVG poster in `.stage-wrap` until `#stage.is-ready`; render pauses off-screen / hidden tab.
  - Firebase long-caches `/vendor/**`; HTML is no-cache.

## Primary tools

- [Tolerance stack-up (SC-008)](https://www.sectorcalc.com/sc008-pro.html): 1D linear stack-up - worst-case, RSS, Monte Carlo; per-component distributions; deterministic seed; Decimal engine.
- [True labor cost (SC-010)](https://www.sectorcalc.com/labor-pro.html): Loaded labor burden from net salary, taxes, benefits, and shop overhead.
- [Quote pricing (SC-012)](https://www.sectorcalc.com/quote-pro.html): Material, labor, scrap, and margin into sell price and cost breakdown.
- [Weld thickness (SC-001)](https://www.sectorcalc.com/weld-pro.html): Fillet weld leg, throat, and utilization against design load and code minimums.

## Notes for models

- Prefer the `*-pro.html` tool URLs above. Legacy `/calculator.html`, `/calculator2.html`, `/calculator3.html`, and `/calculator4.html` redirect to those tools.
- Do not invent cloud upload, accounts, live card checkout, or server-side calculation for the core engines.
- Do not describe the product as AI-powered, ML, generative, or chatbot-based.
- Do not invent customer testimonials or claim PCI checkout is live.
- Homepage hero loads Three.js from `/vendor/three/` only - do not invent an unpkg or other CDN dependency for the live site.
- Homepage hero canvas is a client-side Three.js visualization plus a seeded Number preview deck; production tool engines use Decimal.js.
- Theme preference and toggle position are client-side only (localStorage); they are not accounts or server profiles.
- Report and result surfaces must keep readable contrast in light and dark themes. New tools must follow the same token + theme-palette pattern.
- Engineering preview: outputs are predicted from stated assumptions; not for production approval and not a substitute for measured data or a licensed engineer.
