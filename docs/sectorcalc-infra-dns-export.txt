  match /{document=**} { allow read, write: if false; }  → EVET, default deny var.
  Her koleksiyon ayrı açılıyor; wildcard ile açık erişim YOK.

KOLEKSİYON MATRİSİ:

  Koleksiyon                    | Client READ              | Client WRITE
  ------------------------------|--------------------------|-----------------------------
  leadIntents/{id}              | admin claim              | create (validated fields)
  leadIntents/{id}/activity     | admin claim              | deny (CF Admin SDK)
  betaPartners/{id}             | admin claim              | create (validated)
  benchmarkSubmissions/{id}     | admin claim              | create (validated)
  reportFeedback/{id}           | admin claim              | create (validated)
  calculator_feedback/{id}      | admin claim              | create (validated)
  toolFeedback/{id}             | admin claim              | update status (admin only)
  users/{uid}                   | own uid                  | DENY (subscription CF yazar)
  users/{uid}/purchases/{id}    | own uid                  | DENY
  premiumEntitlements/{id}      | own userId match         | DENY (webhook yazar)
  reports/{id}                  | owner uid                | create (own uid); update/delete DENY
  approvedReports/{id}          | public_verify visibility | DENY (Admin SDK only)

ADMIN TANIMI:
  request.auth.token.admin == true

SUBSCRIPTION / LEAD KURALI:
  - Client users/{uid} yazamaz → subscription yalnız stripeWebhook (Admin SDK)
  - Lead pipeline update → updateLeadPipeline Cloud Function (Admin SDK)
  - Admin panel read-only Firestore; write'lar CF üzerinden
