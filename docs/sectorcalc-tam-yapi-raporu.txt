7.1 AKIL 2 — REQUIREMENT / CONSTRAINT ENGINE
Soru: Verilen hedef ve bilinen inputlarla hangi inputlar eksik, türetilebilir,
      varsayılanlanabilir veya uzman girdisi gerektirir?

Kök dizin: src/lib/formula-governance/requirement-engine/

Ana fonksiyon: solveRequiredInputs({ ontology, goalId, knownInputs })
Çıktı: RequirementSolveResult
  - status: ready_to_calculate | need_more_data | blocked
  - requiredMissingInputs
  - optionalRecommendedInputs
  - advancedRecommendedInputs
  - defaultedInputs
  - derivedResolutionPlan
  - selectedFormulaPath
  - blockers, warnings

Algoritma:
  1. Goal bul (ontology.goals)
  2. Yapısal kontrol: detectCircularDependencies, detectUnreachableTargetVariables
  3. Formula adayları: getAcceptedFormulaCandidatesForGoal()
  4. Her aday için: resolveRequirementsForFormulaPath()
  5. Path skorlama: scoreRequirementPath()
  6. En iyi path seç, önerileri topla

7.2 AKIL 1 — VALIDATION / ORACLE / AUDIT ENGINE
Soru: Inputlar ve sonuçlar matematiksel, fiziksel ve iş mantığı açısından geçerli mi?

Kök dizin: src/lib/formula-governance/runtime-validation/

Ana fonksiyon: validateCalculationInputsAndResult()
Kontroller:
  1. checkAllBoundaries() — min/max/nonNegative/integer
  2. checkDimensionConsistency() — boyut kuralları
  3. evaluateInvariants() — formula node invariant'ları
  4. validateContractDerivedTargets() — currency non-negative, percent 0–100

Oracle: src/lib/formula-governance/oracle/ (26 dosya)
  Bağımsız referans modeller → ORACLE_COMPARISON_PASS/FAIL

Audit runner: src/lib/formula-governance/audit-runner.ts
  Çıktı: PASS | NEEDS_REVIEW | FAIL | DISABLE_OR_SOFTEN

BÖLÜM 8 — CALCULATION ONTOLOGY & GRAF YAPISI
================================================================================

Kök: src/lib/formula-governance/calculation-ontology/

8.1 TEMEL TİPLER (ontology-types.ts)

  CalculationVariable:
    id, label, role, dimension, unit, knowledgeLevel
    constraints, defaultValue, missingRisk, requiredForOutputs

  FormulaNode:
    output, requiredInputs, optionalInputs
    expression, invariants, inverseMappings

  CalculationGoal:
    targetVariable, acceptedFormulaNodes, decisionGoal

  CalculationOntology (kök aggregate):
    slug, sector, variables[], formulas[], goals[]
    defaultAssumptions, professionalUpgradeVariables

  DependencyEdge / DependencyGraph:
    Variable-to-variable kenarlar (formula node üzerinden)

  VariableRole: input | derived | constant | target
  VariableKnowledgeLevel: user_known | estimable | defaultable | system_derived | expert_required
  VariableDimension: currency | percent | count | length | ... | dimensionless

8.2 FORMULA GRAPH (formula-graph.ts)
Formula seviyesinde lookup/topoloji:

  getFormulaById(ontology, id)
  getFormulaByOutputVariable(ontology, variableId)
  getFormulaCandidatesForOutput(ontology, variableId)
  getAcceptedFormulaCandidatesForGoal(ontology, goal)
  topologicalFormulaOrder(ontology) — topological sort

8.3 DEPENDENCY GRAPH (dependency-graph.ts)
Değişken seviyesinde kenar indeksi:

  buildDependencyGraph(ontology) → { variableIds, edges, formulaByOutput }
  getDependenciesForTarget(ontology, targetVariableId) — BFS upstream
  getDerivedVariableChain(ontology, variableId)
  detectCircularDependencies(ontology) — DFS cycle detection
  detectUnreachableTargetVariables(ontology)

İlişki:
  Formula graph = node seviyesi (formül düğümleri)
  Dependency graph = değişken seviyesi kenarlar (formül inputlarından türetilir)
